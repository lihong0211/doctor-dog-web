# EN `/english` 暗色学习工作台设计规格

## 目标

将 `apps/en` 的 `/english` 页面改造为与 `apps/ai` 同系的暗色技术工作台，统一使用深色画布、翠绿色强调色、克制层级和紧凑的信息密度，同时保持现有路由、接口、权限、表格和编辑流程不变。

## 范围

本次只改造 `/en/english`。

包含以下六个现有模块：

- 单词管理
- 词库管理
- 用户管理
- 词根管理
- 词缀管理
- 生活口语

不改造以下区域：

- `/store` 商城及其登录、注册和后台页面
- BusinessData 页面
- 独立 EnDesktop 管理路由
- 后端接口和数据结构

## 视觉方向

沿用 AI 项目已经批准并上线的设计语言：

- 主画布：`#07090D`
- 一级表面：`#0D1117`
- 二级表面：`#131922`
- 三级表面：`#19212C`
- 常规边框：`#232D39`
- 强边框：`#344150`
- 主文字：`#F3F6F8`
- 次要文字：`#AAB5C0`
- 弱文字：`#6F7C89`
- 主强调色：`#00C98D`
- 悬停强调色：`#18D9A1`
- 弱强调底色：`rgba(0, 201, 141, 0.12)`

字体继续使用项目现有字体栈，不引入新的运行时视觉依赖，不复制第三方品牌素材。

## 页面架构

### 共用框架

保留现有 `ProLayout`、`PageContainer`、路由出口和登录状态组件。

调整框架视觉：

- 顶栏和菜单使用暗色表面。
- 当前导航使用翠绿文字或指示线。
- 内容画布使用统一深色背景。
- `/english` 内容区域取消旧式浅色后台画布。
- 商城及其他路由不挂载 `/english` 专属样式。

### `/english` 页面头部

页面顶部增加紧凑的学习工作台头部：

- 辅助标识：`ENGLISH WORKBENCH`
- 主标题：`英语学习中心`
- 简短说明：管理单词、词库、词根词缀和生活口语内容。
- 右侧可显示当前模块名称，不增加新的业务统计请求。

头部只承担定位和层级表达，不增加新功能。

### 模块导航

六个现有模块改为左侧导航，不改变模块组件本身的数据请求和编辑逻辑。

导航顺序固定为：

1. 用户
2. 单词
3. 词库
4. 词根
5. 词缀
6. 日常用语

视觉与响应式调整：

- 桌面端使用固定左侧栏，右侧显示当前模块。
- 平板端缩窄侧栏，但保留图标和文字。
- 手机端隐藏固定侧栏，通过明确标注的按钮打开 Drawer。
- 当前项使用翠绿文字、弱强调背景和左侧指示。
- 未选中项使用次要文字。
- 焦点状态清晰可见，支持键盘操作。

### 内容工作区

当前模块内容置于暗色工作区面板：

- 面板背景使用二级表面。
- 1px 常规边框。
- 8px 圆角。
- 不使用厚重阴影、渐变和发光效果。
- 内容区域自适应剩余高度，表格内部滚动。

## 组件规范

### ProTable 和 Table

- 表头使用一级或三级表面。
- 表体使用二级表面。
- 行悬停使用三级表面。
- 分割线使用常规边框。
- 固定列和滚动阴影必须与暗色背景一致。
- 分页、筛选、密度和刷新按钮使用暗色控件状态。

### Form、Input、Select

- 输入背景使用一级表面。
- 边框使用强边框。
- 聚焦边框使用主强调色。
- 占位文字使用弱文字。
- 禁用状态保持可读，不使用纯白底。

### Card、Drawer、Modal

- Card 和弹层内容使用二级表面。
- Drawer 和 Modal 遮罩沿用 Ant Design 行为。
- 标题、关闭按钮、底部操作区使用统一暗色层级。
- 不改变现有表单字段、提交逻辑和关闭行为。

### Button、Tag、反馈状态

- 主按钮使用 `#00C98D`。
- 危险操作保留红色语义。
- 链接按钮使用翠绿或主文字。
- Tag 使用低饱和暗色底，不使用大面积浅色填充。
- 成功、警告、错误消息保留原有业务语义。

## 响应式与可访问性

- 桌面目标视口：1440×1024。
- 平板目标视口：1024×768。
- 手机目标视口：390×844。
- 390px 宽度下页面不得出现整体横向滚动。
- 手机端侧栏通过 Drawer 展示，不占用内容宽度。
- 宽表格可在工作区内部横向滚动。
- 所有交互控件保留 `:focus-visible`。
- 遵守 `prefers-reduced-motion`。
- 保持 Ant Design 组件现有的键盘语义和 ARIA 行为。

## 技术方案

### 主题入口

在 `apps/en/src/theme/` 建立 EN 项目的集中主题：

- `tokens.ts` 定义语义设计令牌。
- `antdTheme.ts` 映射 Ant Design 和 Pro Components 主题。
- `main.tsx` 或 `App.tsx` 使用单一 `ConfigProvider` 入口。

主题只改变视觉令牌，不改变路由或业务数据流。

### 样式隔离

- 全局 CSS 负责主题变量、基础画布、焦点和 reduced-motion。
- `/english` 根节点使用专属类名，例如 `.en-workbench`。
- English 页面专属布局写入独立 CSS。
- 对 ProTable、Tabs、Drawer、Modal 的覆盖必须限定在 EN 应用或 `/english` 工作台范围内。
- 不通过全局选择器改变 `/store` 的页面布局。

### 组件边界

- `Root`：保留路由框架，只接入主题化类名和框架视觉。
- `English`：负责工作台头部、左侧导航、移动端 Drawer 和内容面板。
- 六个既有模块：保留数据请求和编辑逻辑，仅移除冲突的浅色视觉硬编码。
- 不创建超出本次需求的通用组件库。

## 测试与验收

### 自动测试

新增测试覆盖：

- 主强调色为 `#00C98D`。
- 画布色为 `#07090D`。
- Ant Design 主题启用暗色算法。
- `/english` 渲染工作台标题。
- 六个模块导航仍然存在且顺序正确。
- 默认模块为“用户”。
- English 专属 CSS 包含响应式、焦点和 reduced-motion 规则。
- `/english` 相关文件不再包含根级浅色背景。

### 构建验证

必须执行：

```bash
pnpm --dir apps/en test
pnpm --dir apps/en build
```

如果项目当前没有测试脚本，则增加最小 Vitest 测试基础，仅用于主题和页面结构回归，不引入 Storybook。

### 视觉验收

至少抽查：

- 单词管理
- 词库管理
- 词根或词缀管理
- 一个新增/编辑弹窗

分别检查桌面、平板和手机视口。

## 发布

改造完成并验证后：

1. 提交源码和 `apps/en/dist`。
2. 推送 `main` 到远端。
3. 执行仓库现有 `deploy/deploy.sh`。
4. 确认服务器 `git pull` 和 `nginx -t` 成功。
5. 检查生产地址 `/en/english` 返回 HTTP 200，并加载本次构建的新资源。

- 不修改后端接口。
- 不改商城和后台页面。
- 不增加新的业务功能、统计接口或权限规则。
- 不升级 React、Ant Design、Pro Components 或构建工具版本。
