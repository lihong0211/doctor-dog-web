# LLM、VLM 与 Agent 面试题双版本整理方案

## 目标

将用户提供的面试题与补充的工程实战题合并、去重并重新分类，生成两份可独立使用的中文 Markdown 文档：

1. 完整版：用于系统学习和查漏补缺。
2. 高频速查版：用于面试前快速复习和口述训练。

## 输出文件

- `LLM_VLM_Agent_面试题完整版.md`
- `LLM_VLM_Agent_面试高频速查.md`

文件保存在当前工作目录。

## 完整版结构

按以下主题组织：

1. LLM 基础原理与架构
2. 训练、数据与微调
3. 对齐、RLHF 与偏好优化
4. 推理优化与部署
5. VLM 与多模态
6. RAG 与搜索系统
7. Agent、工具调用与多智能体
8. 模型、RAG 与 Agent 评估
9. 安全、隐私与生产治理
10. 项目设计、线上排障与连续追问
11. 技术趋势与开放讨论

每题采用以下适用字段：

- **问题**
- **参考答案**：适用于存在相对稳定技术结论的问题。
- **答题框架**：适用于项目题、观点题和开放题。
- **常见追问**：用于训练连续深挖能力。
- **易错点**：只在存在典型误区时添加。

## 高频速查版结构

精选约 80–100 道最能区分候选人水平的问题，沿用完整版主题顺序。答案压缩为适合 1–3 分钟口述的版本，并使用以下标记：

- `★★★`：高频且必须掌握。
- `★★`：常见进阶题。
- `★`：岗位相关题。

速查版不简单复制完整版长答案，而是提供：

- 一句话结论；
- 3–5 个核心要点；
- 一个关键权衡或易错点；
- 必要时给出追问方向。

## 内容原则

1. 合并语义重复的问题，保留信息更完整的表述。
2. 不编造项目经历、业务指标或个人观点。
3. 对项目题提供可替换的 STAR/指标化答题框架。
4. 对存在实现差异或快速演进的技术，避免写成唯一结论。
5. 公式只保留面试中有解释价值的部分，并同时给出直观含义。
6. 回答强调“原理、选型、权衡、排障、指标”，避免只罗列名词。
7. 文档独立可读，不要求读者回看原始附件。

## 质量检查

生成后执行以下检查：

1. 检查标题层级和目录结构。
2. 搜索 `TODO`、`TBD` 和空答案。
3. 检查两份文档均涵盖全部核心主题。
4. 抽查关键技术题的准确性和前后表述一致性。
5. 检查速查版题目数量是否处于约定范围。
6. 确认没有覆盖或修改用户已有文件。

## 非目标

- 不生成针对某一家公司的内部题库。
- 不提供虚假的“标准项目数据”。
- 不把开放观点题包装成唯一正确答案。
- 不展开为教材级推导或论文逐段解读。
